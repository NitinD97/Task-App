The following is the list of API requests:

Create a User

POST /users

This request creates a user and returns a token as well as created user details. The body should be as follows, with type as JSON:
{ "name":"username", "email": "example@mail.com", "password": "1111111" }

Login User

POST /users/login

This request logs in a user and returns a token as well as user details. The body should be as follows, with type as JSON:
{ "email": "nitin@mail.com", "password": "1111111" }

The body should be as follows, with type as form-data and in key-value pair:
avatar: file.png

Get all tasks

GET /tasks

This request gets all the tasks updated by the user. The header should contain token received while creating/logging in the user:
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJfaWQiOiI1ZWEzZjA5M2IxYWQ2NjI4NzQ0YTEzNjgiLCJpYXQiOjE1ODc4MDIyNTl9.qAdhkdGXCtHkrSYzai9n3950LY7vvvZ34xesclw-m9d


The tasks can be sorted ascending/descending, at any field from all the available fields for a tasks as follows:
"_id" ,"description", "createdAt", "completed"
by giving query param as
/tasks?sortBy=<fieldname>_<order> as /tasks?sortBy=completed_desc

Pagination can also be used by sending limit and skip params, both the params should be provided.
/tasks?limit=10&skip=10.
